# UI 設計說明

## 1. 設計原則

### 1.1 核心目標
- **減少點擊**：優先使用單選按鈕而非下拉選單
- **清晰易懂**：字段標籤使用中文，必要時附英文名稱
- **即時反饋**：輸入錯誤時立即提示，不等到提交
- **一致性**：與現有 LIS7 POS52 年結表模板保持一致

### 1.2 可用性要求
- 單一查詢操作不超過 5 次點擊
- 必填字段有明顯標記（*）
- 錯誤提示具體明確（例如「年齡必須在 18-80 之間」而非「輸入無效」）

## 3. 交互規範

### 3.1 表單驗證

| 驗證類型 | 觸發時機 | 提示方式 |
|----------|----------|----------|
| 必填字段 | 失去焦點時 | 紅色邊框 + 提示文字 |
| 日期格式 | 失去焦點時 | 紅色邊框 +「日期格式無效」 |
| 年齡範圍 | 失去焦點時 | 紅色邊框 +「年齡必須在 18-80 之間」 |
| 金額格式 | 失去焦點時 | 紅色邊框 +「請輸入有效數字」 |

### 3.2 加載狀態
- 計算過程中顯示 loading 動畫
- 超過 2 秒顯示進度提示
- 禁止重複提交

### 3.3 錯誤處理
- 網絡錯誤：「連接服務器失敗，請稍後重試」
- 計算錯誤：彈窗顯示具體錯誤原因
- 權限錯誤：「您沒有權限執行此操作」

---

## 4. 響應式設計

### 4.1 斷點
- 桌面端：≥1440px（主要使用場景）
- 平板端：768px-1439px（不支持，但可瀏覽）
- 手機端：<768px（不支持）

### 4.2 適配策略
- 由於是內部系統且不支持移動設備，主要優化桌面端體驗
- 表格在較小屏幕上可橫向滾動

---

## 5. 無障礙設計

- 所有輸入框有明確的 label
- 錯誤提示使用顏色 + 文字（不只靠顏色）
- 鍵盤可操作（Tab 鍵導航）
- 對比度符合 WCAG 2.0 AA 標準
